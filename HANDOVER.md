## 4. 동작 구조

### 4.1 화면 흐름 (4단계)

```
① PRESS START (VCR 파란 화면)
      ↓ CRT 꺼짐 (0.6s)
② 테이프 삽입 + 부팅 리드아웃 (2.6s)
      ↓
③ 데스크톱 = 디텍션 벤치  ←── 사이트의 본체
      ↓ 아이콘/시작메뉴 클릭
④ 문서 창 (ABOUT / CONTROL / AI_REEL / STUDIO / CONTACT)
```

**중간 영상 재생 단계는 없습니다.** 초기에 계획했으나 클라이언트 요청으로 제외했습니다. 완성 영상은 데스크톱의 `AI_REEL.AVI` 창 안에서 재생됩니다.

### 4.2 테이프 삽입 (`styles.css` → `THE TAPE` 섹션)

진짜 3D입니다. `perspective: 820px` 스테이지 안에서:
1. 아래에서 비스듬히 날아 올라옴
2. **정면으로 서서 정지** — 라벨을 읽는 구간
3. **`rotateX(90deg)`로 눕음** — 라벨이 위로, 등(spine)이 카메라로
4. 납작한 채 **길이 방향으로 슬롯에 밀려 들어감**

슬롯(`.vcr-slot-lip` / `-throat` / `-flap`)이 실제로 존재하고, 테이프는 z-index가 아니라 **3D 깊이 정렬**로 슬롯 뒤에 가려집니다. 축소는 `scale`이 아니라 **원근이 자동으로** 합니다.

> ⚠️ 6면체 면 배치는 반드시 `translate(-50%,-50%) rotateX(90deg) translateZ(110px)` 중앙정렬 방식이어야 합니다. `transform-origin` 방식으로 바꾸면 모서리가 안 맞아 육면체가 안 닫힙니다. (실제로 한 번 깨뜨렸다가 되돌렸습니다.)

### 4.3 데스크톱 = 디텍션 벤치 (핵심)

3컬럼: **아이콘 레일 | 씬 뷰포트 | 컨트롤 패널**

이게 이 사이트의 핵심입니다. *"우리는 객체를 제어합니다"* 라고 **말하는 대신 보여줍니다.**

**탐지 대상 12개** (`main.js` → `STATIC_TARGETS`)
전부 그림 안에 **실제로 존재하는 것들**이고, 좌표는 이미지에 박스를 다시 그려서 눈으로 검증했습니다. 빈 도로 위에 떠 있는 박스는 하나도 없습니다.

```
CAR 1001 (주차된 청록차) · PERSON 203/204/205 · LIGHT 07/08 (신호등)
HYDRANT 12 (소화전) · BENCH 04 · POSTBOX 11 · BOX 09 (우체통)
SIGN 02 (Z 간판) · TREE 05 (가로수)
```

person / car / traffic light / fire hydrant / bench는 **전부 실제 COCO 클래스**입니다. 진짜 탐지 모델이 이 프레임에서 반환할 법한 결과입니다.

`kind: "subject"`(사람) / `kind: "object"`(나머지) 로 나뉩니다. **이게 회사가 파는 구분 그 자체**라, 캡션 없이도 화면이 피치를 말합니다.

**슬라이더 4개는 전부 실제로 작동합니다.** 움직이지 않는 슬라이더는 그냥 가짜 스크린샷이고, taste-skill이 명시적으로 금지하는 항목입니다.

| 슬라이더 | 실제 동작 |
|---|---|
| **Object Detection Sensitivity** | confidence 임계값. 낮추면 타겟이 탈락 |
| **Motion Tracking Precision** | 트래커가 진실을 쫓는 강도. §4.4 참고 |
| **Face Recognition Accuracy** | subject에만 얼굴 락. 밤엔 더 높아야 함 |
| **Scene Relight** | 낮→밤 그레이딩. §4.5 참고 |

버튼: `Start/Stop AI Feed`(탐지 on/off), `Apply Controls`(confidence 재측정 → 박스 재구성), `Reset Parameters`.

### 4.4 트래커 (`main.js` → `updateTracker`)

**박스는 물체의 진짜 위치가 아닙니다.** 진짜 위치를 쫓아가는 **추정치**입니다. 이게 Motion Tracking Precision이 패널에 있을 자격을 얻는 지점입니다.

```js
follow = 1 - (1 - (0.02 + p*0.55))^(dt*60)   // p = precision/100
```

- Precision 높음 → 박스가 물체에 용접된 듯 붙음
- Precision 낮음 → **뒤처지고, 지나치고, 흔들림**

시안색 궤적선은 **물체의 경로가 아니라 트래커가 추정한 경로**입니다. 엉성한 트래커는 엉성한 궤적을 그립니다 — 그게 보여주려는 것입니다.

### 4.5 카메라 무브 (`main.js` → `advanceCamera`)

정지 이미지 위에서는 트래커가 아무것도 증명하지 못합니다. 그래서 **카메라가 움직입니다** (느린 팬 + 푸시인).

- 합성 지오메트리 **0** — 2D 유사변환(scale + translate) 하나뿐
- CSS가 `.scene-plate`에 거는 변환과 JS `truthBox()`가 박스에 거는 계산이 **동일한 식**이라 어긋날 수 없음
- **Camera는 회사가 파는 네 축 중 하나**라 메시지에도 맞음

**컨트롤을 만질 때만 움직입니다** (`engage()` → `driveUntil`). 가만두면 프레임을 붙잡고 정지(`FEED HELD`), 마지막 입력 후 0.9초 코스팅하다 멈추고, 트래커가 자리잡으면 rAF 루프도 정지합니다. **유휴 애니메이션 루프가 없습니다.**

### 4.6 Scene Relight (낮 ↔ 밤)

두 겹입니다. 둘 다 `.scene-plate` 안에 있어서 **카메라를 같이 탑니다** (안 그러면 카메라가 팬할 때 가로등만 제자리에 붙어 있어 즉시 들통납니다).

1. **멀티플라이** 짙은 남색 그라디언트 → 원래 어두운 곳은 그대로, 밝은 곳이 색을 먹음
2. **스크린** 실용등 7개 (신호등 2, 가로등, 상점 유리들, 전광판). 위치는 그림에서 직접 딴 좌표

**밤에는 탐지가 실제로 어려워집니다.** 어두울수록 confidence가 최대 **24점** 깎이고(`NIGHT_CONFIDENCE_COST`), 얼굴 락 임계값이 60→82로 올라갑니다(`NIGHT_FACE_COST`). 박스를 되찾으려면 **Sensitivity를 올려야 합니다.** 슬라이더 두 개가 실제로 싸웁니다.

### 4.7 렌더 루프

**rAF 루프는 하나뿐입니다** (`loop()` + `active` Set + `startLayer`/`stopLayer`). 캔버스마다 따로 돌리면 한 프레임에 화면을 여러 번 다시 그리게 됩니다. 노이즈는 1/3 프레임레이트로 스로틀 — 실제 테이프 노이즈는 60fps가 아닙니다. `visibilitychange`로 탭이 숨으면 정지합니다.

## 5. `scene.png` — 만든 과정과 교체 방법

**현재:** 372×179, 63KB.

원본은 클라이언트가 준 **2976×1432 / 6.7MB** 영상 스크린샷이었습니다. 처리 과정:

1. **마우스 커서 제거** — 도로 한가운데 박혀 있었음. 깨끗한 도로에서 패치를 떠다 덮음
2. **AI 생성 워터마크 제거** — 우하단 반짝이. 연석 대각선을 따라 위상이 맞는 곳에서 패치를 뜸
3. **372×179로 면적평균 다운스케일** — 영상 압축으로 도트 격자가 뭉개져 있었는데(격자 정렬도 lift 1.1 = 사실상 없음), 다운스케일하니 **오히려 도트가 또렷해짐**
4. 64색 팔레트 양자화

**결과: 6.7MB → 63KB (107배), 화질은 향상.**

풀해상도 정리본은 `assets-src/scene-source.png`에 백업 (gitignore됨).

### 그림을 교체하려면

1. 새 그림을 `public/scene.png`에 저장 (**소문자 필수** — Vercel은 리눅스라 대소문자를 가립니다)
2. `main.js`의 `PLATE_W` / `PLATE_H` 를 새 크기로 수정
3. `STATIC_TARGETS`의 12개 좌표를 새 그림에 맞게 재측정
4. `styles.css`의 `.scene-lamps` 라디얼 그라디언트 위치 재조정

> 💡 좌표 검증은 **반드시 이미지에 박스를 그려서 눈으로 확인**하세요. 눈대중은 반드시 어긋납니다. Python + Pillow로 `public/scene.png`를 3배 확대하고 박스를 오버레이하는 스크립트를 쓰면 5분이면 됩니다.

## 7. ⚠️ 다시 시도하지 말 것

### 7.1 그림 위에 합성 자동차 그리기 — **실패, 폐기**

*"교차로가 비었으니 차를 그려 넣자"* 는 자연스러운 발상이고, 실제로 두 번 시도해서 두 번 다 어색하게 나왔습니다. 세 번째도 마찬가지일 겁니다.

**근본 원인: 이 그림은 손으로 그린 픽셀아트라 원근이 수학적으로 일관되지 않습니다.**

세 가지 독립적인 방법으로 도로 방향을 측정했고, 셋이 서로 모순됩니다:

| 측정 방법 | 결과 |
|---|---|
| 교차로 네 모서리(연석) 기하 | 도로1 = **46°** |
| 손으로 그은 차선 | 도로1 = **26°** |
| 그림 속 주차된 차의 축 | 도로2 = **-25°** (연석 측정은 -11°) |

횡단보도 줄무늬 **55개**를 추출해 소실점을 최소제곱 피팅한 결과 **잔차 12~30px**. 일관된 소실점이 존재하지 않습니다.

시도했다가 버린 것들:
- 4점 호모그래피 (교차로 모서리 기준) → 소실점이 뒤집힘
- 줄무늬 기반 소실점 추정 → 잔차 과다
- 도로 축 2개로 평행사변형 바닥면 만들기 → 차가 도로를 안 지킴

**대신 카메라를 움직입니다** (§4.5). 합성 지오메트리가 0이라 어색해질 수 없고, 메시지에도 맞습니다.

**정말 자동차가 필요하다면** 유일하게 안전한 길: **그림 속 주차된 차를 스프라이트로 오려내서** 도로를 따라 움직이는 것. 작가가 직접 그린 차라 이 그림의 투영과 100% 일치합니다. 색만 바꿔 2~3대로 늘리면 됩니다. (원래 자리는 인페인팅 필요)

### 7.2 씬 뷰포트에 `min-height: 0` 주기 — **버그의 원인**

모바일에서 씬이 통째로 사라졌던 원인입니다. 세로 스택에서 컨트롤 패널이 높이를 다 먹으면 `1fr` 행에 남는 게 0px이 되고, 최소 높이가 없으면 저항 없이 사라집니다.

지금은 `min-height: 150px` 안전장치가 있습니다. **넘치는 건 복구 가능하지만 0px로 사라지는 건 눈치채기도 어렵습니다.** 이 가드를 제거하지 마세요.

### 7.3 아이콘 레일에 `flex-wrap` 쓰기 — **버그의 원인**

줄바꿈된 flex 줄이 자기 전체 높이를 grid 행에 제대로 보고하지 않아서, `CONTACT.COM`이 3번째 줄로 밀리면 씬이 그 위를 덮었습니다. **모바일 레일은 반드시 `display: grid`** 여야 합니다.

## 8. 알아둘 함정들

- **창 높이 상한 92%** — `.win95-window`가 `top: 46%` 중앙정렬이라, 높이가 92%를 넘으면 **타이틀바가 화면 위로 잘려 나갑니다.**
- **미디어 쿼리 순서** — `1100px → 900px → 640px` 순서여야 합니다. 뒤에 오는 게 이깁니다.
- **파일명 소문자** — 로컬 Windows는 대소문자를 안 가리지만 Vercel(리눅스)은 가립니다. `Scene.PNG`로 저장하면 로컬만 되고 배포는 깨집니다.
- **`git commit` 메시지** — Bash에서 PowerShell 히어스트링(`@'...'@`)을 쓰면 깨집니다. heredoc(`<<'MSG'`)을 쓰세요.
